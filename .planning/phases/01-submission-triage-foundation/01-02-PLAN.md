<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Anonymous user → /api/auth/register/ | untrusted self-registration input |
| Registered-but-unapproved user → feature routes | privilege boundary enforced by approval gate |
| Admin → approve/reject | privileged actions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-06 | Elevation of Privilege | Client sets role=admin during registration | mitigate | Server forces role to student/lecturer in serializer; admin/kaprodi only via seed_admin (Task 1) |
| T-1-07 | Elevation of Privilege | Unapproved user reaches feature routes | mitigate | IsApprovedUser on feature endpoints + requireAuth loader redirects to /pending-approval (D-20, Pitfall 8) |
| T-1-08 | Information Disclosure | Lecturer dropdown leaks unapproved lecturers | mitigate | /api/users/lecturers/ filters is_approved=True (Pitfall 7); test asserts it |
| T-1-09 | Spoofing | Registration without CSRF | mitigate | Inherits CsrfViewMiddleware + X-CSRFToken from Plan 01 client |
| T-1-10 | Tampering | Duplicate NIM/email account takeover attempt | mitigate | unique constraints + serializer validation with explicit errors (Task 1) |

All threats mitigated. No HIGH-severity unmitigated threat.
</threat_model>
