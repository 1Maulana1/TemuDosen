## Spacing Scale

Declared values (multiples of 4 only):

| Token | px value | Tailwind class | Usage |
|-------|---------|----------------|-------|
| xs | 4px | p-1 / gap-1 | Icon gap, inline badge padding |
| sm | 8px | p-2 / gap-2 | Chip padding, compact list gaps |
| md | 16px | p-4 / gap-4 | Default form element spacing, card body padding |
| lg | 24px | p-6 / gap-6 | Section padding, sidebar header padding |
| xl | 32px | p-8 / gap-8 | Layout gutters on desktop |
| 2xl | 48px | py-12 | Major section breaks |
| 3xl | 64px | pb-16 | Bottom padding to clear sticky footer |

**Exceptions:**
- Touch targets: minimum 44√ó44px enforced on all interactive elements (DESIGN.md WCAG requirement). Use `min-h-[44px] min-w-[44px]` when a button would otherwise be smaller.
- Sticky header height: `h-14` (56px) on mobile submission form; `h-16` (64px) on dashboards.
- Progress bar height: `h-1.5` (6px) ‚Äî cosmetic only, not a spacing token.
- Scrollbar width: 4px (mobile), 6px (admin desktop) ‚Äî scoped CSS, not Tailwind spacing.

## Typography

All font sizes are in pixels. "rem" values use 16px base.

| Role | Size | Tailwind | Font Family | Weight | Tailwind weight | Line Height |
|------|------|----------|-------------|--------|-----------------|-------------|
| Caption / label | 11px | `text-[11px]` | Public Sans | 400 (regular) | `font-normal` | 1.4 |
| Body / form label | 14px | `text-sm` | Inter | 400 (regular) | `font-normal` | 1.5 |
| Body emphasis / button | 14px | `text-sm` | Inter | 700 (bold) | `font-bold` | 1.5 |
| Section heading | 18px | `text-lg` | Plus Jakarta Sans | 700 (bold) | `font-bold` | 1.3 |
| Page heading / display | 24px | `text-2xl` | Plus Jakarta Sans | 700 (bold) | `font-bold` | 1.2 |

**Rules:**
- Minimum body font: 14px (`text-sm`) ‚Äî never below this on any user-facing text (DESIGN.md).
- Labels inside form fields use `text-sm font-bold text-slate-800` (14px, bold, near-black).
- Helper/hint text uses `text-[11px] font-normal text-gray-500` (11px, regular, gray).
- Buttons use `text-sm font-bold` (14px, 700).
- Do NOT use `font-medium` (500), `font-semibold` (600), or `font-extrabold` (800) ‚Äî only `font-normal` and `font-bold` are permitted.
- The `font-body`, `font-headline`, `font-label` Tailwind aliases must be used; never hardcode font-family inline.

Elevated cards (summary/dark): `bg-slate-900 rounded-2xl p-4 text-white shadow-lg overflow-hidden relative`

## Accessibility Contract

All elements must meet WCAG 2.1 AA. Specific requirements for Phase 1:

- Touch targets: all interactive elements minimum `min-h-[44px] min-w-[44px]`
- Color-only status: every status badge carries both color AND text label (never color alone)
- Focus-visible: `focus-visible:ring-2 focus-visible:ring-primary focus-visible:outline-none` on all interactive elements
- Form labels: every `<input>` and `<select>` has an associated `<label>` (not placeholder-only)
- Error messages: linked to their input via `aria-describedby`
- File upload: `<input type="file" accept=".pdf">` with visible label, not icon-only
- Required fields: marked with `*` (`text-error`) AND `aria-required="true"`
- Loading states: `aria-busy="true"` on button, `aria-live="polite"` on result region
