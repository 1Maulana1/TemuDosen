<objective>
Establish the Walking Skeleton: scaffold the Django backend and React (Vite) frontend, define the
multi-role `CustomUser` model and set `AUTH_USER_MODEL` BEFORE the first migration, wire the
session + CSRF authentication handshake end-to-end (login → authenticated `me` → role-gated React
route), seed the admin/kaprodi accounts, and lay down the full test scaffolding (Wave 0 gaps from
VALIDATION.md). All subsequent Phase-1 feature plans build on this foundation.

Purpose: De-risks the CORS + session-cookie + CSRF integration — the single most failure-prone part
of a DRF + React SPA — before any business logic is written. Establishes the directory layout,
settings split, router map, and API client that every later plan depends on. Implements D-21
(server-side sessions, not JWT) and D-25 (seeded admin/kaprodi).

Output: A runnable two-repo project where the seeded admin logs in through the UI and lands on a
role-gated route, with green backend + frontend test scaffolds.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| React SPA → Django API | All user input crosses here; untrusted until validated server-side |
| Browser cookie store → Django session | sessionid + csrftoken cookies authenticate every request |
| package installs (pip/npm) → build | supply-chain surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Tampering | AUTH_USER_MODEL migration ordering | mitigate | CustomUser + AUTH_USER_MODEL defined before first makemigrations/migrate (Task 1); `makemigrations --check` gate |
| T-1-02 | Spoofing | Session login without CSRF | mitigate | CsrfViewMiddleware + X-CSRFToken header from non-HttpOnly csrftoken cookie; GET /api/csrf/ on mount (Tasks 2,3) |
| T-1-03 | Spoofing | Cross-origin session cookie not sent | mitigate | client.ts forces credentials:'include'; Vite proxy makes requests same-origin in dev (Task 2,3) |
| T-1-04 | Elevation of Privilege | Unauthenticated access to /api/auth/me/ | mitigate | IsAuthenticated; test asserts 403 for anonymous (Task 2) |
| T-1-05 | Information Disclosure | Session cookie theft via JS | mitigate | SESSION_COOKIE_HTTPONLY=True; csrftoken intentionally readable but carries no session (base.py) |
| T-1-SC | Tampering | pip/npm installs | mitigate | All packages [OK]/Approved per RESEARCH Package Legitimacy Audit (no [ASSUMED]/[SUS]/[SLOP]); versions pinned in requirements.txt |

All threats are mitigated. No HIGH-severity unmitigated threat — execution may proceed.
</threat_model>
