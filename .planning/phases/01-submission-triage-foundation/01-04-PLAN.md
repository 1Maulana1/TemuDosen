<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Student → POST /api/submissions/ | untrusted file + form input crosses here |
| Any user → GET /api/files/<uuid>/ | file access boundary — must be ownership-checked (D-29) |
| Uploaded file → disk | untrusted content stored locally |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-15 | Information Disclosure | Direct media file access bypassing auth | mitigate | No MEDIA_URL; /api/files/<uuid>/ requires auth + ownership (student/adviser/admin) per D-29 Pattern 3; tests assert 403 for anon + non-owner (Task 1) |
| T-1-16 | Tampering | Malicious non-PDF disguised as .pdf | mitigate | python-magic magic-bytes check + FileExtensionValidator; file never executed (Task 1) |
| T-1-17 | Denial of Service | Oversized file upload | mitigate | 5MB serializer limit rejecting before disk write; DATA_UPLOAD_MAX_MEMORY_SIZE=6MB transport cap (Task 1, Plan 01) |
| T-1-18 | Information Disclosure | IDOR via guessed/leaked file UUID | mitigate | UUID non-guessable AND ownership check still enforced server-side (Task 1) |
| T-1-19 | Elevation of Privilege | Unapproved or non-student creating submissions | mitigate | IsStudent + IsApprovedUser on POST (Task 1) |
| T-1-20 | Spoofing | Submission POST without CSRF | mitigate | Inherits CsrfViewMiddleware + X-CSRFToken from Plan 01 client |

All threats mitigated. No HIGH-severity unmitigated threat.
</threat_model>
