<objective>
Create the durable storage layer for the logbook pipeline: the `SessionLogbook` Django model (D-05 — a dedicated model, NOT new fields on the overloaded `Session`), the `SessionRecording.duration_seconds` field (D-02), and the `SessionSummary` Pydantic schema (AI-SPEC §4b.1) that validates the LLM's structured output.

Purpose: Every downstream wave writes to or reads from `SessionLogbook`. Getting the field set right now — including the D-11 token/cost tracking fields that the LLM pipeline (Wave 3) populates and the Admin dashboard (Wave 4) sums — prevents rework. This wave lands the model exactly as the locked wave structure requires.

Output: `SessionLogbook` model + migration, `SessionRecording.duration_seconds` migration, `SessionSummary`/`AdvicePoint`/`ImprovementNote` Pydantic schemas with the `no_placeholder_text` validator, and model tests.
</objective>
