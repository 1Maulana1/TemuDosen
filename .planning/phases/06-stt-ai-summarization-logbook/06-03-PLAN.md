<objective>
Build the two external-capability wrappers the async pipeline calls: `services/stt.py` (faster-whisper CPU/int8 transcription, D-01) and `services/summarizer.py` (Anthropic Batch API submission + result parsing, D-03). These are pure service functions with no Celery/task orchestration — that lives in Wave 3 (06-05). Keeping them separate lets each be unit-tested with mocks and keeps the video-provider-agnostic boundary (D-14) clean.

Purpose: Isolate the risky external integrations (a 484MB model that must load once per worker to avoid a documented hang; an async Batch API that must never be polled in a blocking loop) behind tested functions, following the graceful-degradation feature-flag pattern from `calendar.py`.

Output: `stt.py` (worker_process_init model load + transcribe_audio), `summarizer.py` (SYSTEM_PROMPT, TOOL_SCHEMA from SessionSummary, build_batch_request, submit_batch, parse_result), and unit tests that mock `WhisperModel` and the Anthropic client.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| backend → api.anthropic.com | Transcript text (personal data) leaves Indonesia to a foreign processor |
| worker → audio file on disk | Untrusted uploaded audio is decoded by faster-whisper |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05 | Information Disclosure | transcript sent to Anthropic (UU PDP cross-border) | accept | D-03 hybrid design (self-hosted STT, only transcript text to cloud LLM) is the risk-reducing choice already recorded in AI-SPEC §1b; not re-derived here |
| T-06-06 | Denial of Service | oversized/malicious audio decoded by faster-whisper | mitigate | Size/MIME validation is enforced upstream at upload (Phase 5 `CompleteSessionView`, `RECORDING_MAX_UPLOAD_SIZE`) and re-checked by the Wave-3 task before calling `transcribe_audio` — this service does not re-implement validation ad hoc (Security Domain guidance) |
| T-06-07 | Tampering | secrets in logs | mitigate | `_log_error` writes only message + context (logbook_id), never the API key or full transcript body |
</threat_model>
