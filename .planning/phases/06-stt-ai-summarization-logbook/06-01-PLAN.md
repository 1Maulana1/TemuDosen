| Boundary | Description |
|----------|-------------|
| host → Redis broker | If Redis were published to the host, any local process could read/tamper Celery task payloads |
| pip/npm install → build | New third-party packages enter the image |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure / Tampering | Redis broker in docker-compose | mitigate | No `ports:` on the `redis` service — internal Docker network only (Task 3, grounded in 06-RESEARCH.md Security Domain) |
| T-06-SC | Tampering | pip installs (celery/redis/faster-whisper/anthropic) | mitigate | Task 1 blocking-human legitimacy checkpoint before first install; [SUS] verdicts verified as false positives against official registries |
| T-06-02 | Denial of Service | celery-worker concurrency vs CPU cores | accept | `--concurrency=2` conservative default (06-RESEARCH.md Pitfall 5); tunable via env, not a security break |
</threat_model>
